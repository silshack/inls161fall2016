Now that we have added files, let's commit our changes. Committing is sort of like "save as." It tells Git that we want to keep these changes as a new version. This is why we call this task version control.

Let's break this down a bit. The first switch -a tells git to record all the changes made. The second switch -m allows us to enter a commit message. This is a message to ourselves to remind us what we just did. This is very useful for keeping your workflow in order.
